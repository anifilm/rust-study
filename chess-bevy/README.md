# 체스 (Chess) — Bevy

Rust 게임 엔진 [Bevy](https://bevy.org) `0.19` 로 만든 체스 게임입니다.
시작 메뉴에서 **1:1 대전 / AI 대전 / 나가기** 를 선택할 수 있으며,
현재 **1:1 대전(같은 화면에서 두 사람이 번갈아 두는 로컬 모드)** 이 완전히 구현되어 있습니다.

## 실행 방법

```bash
cargo run
```

> 처음 빌드는 Bevy 의존성 컴파일 때문에 몇 분 정도 걸릴 수 있습니다.
> 한글 UI 를 위해 `assets/fonts/NanumGothic-Regular.ttf` 폰트를,
> 기물 그래픽으로 `assets/pieces/*.png` 이미지를 사용합니다.
> (에셋 경로 때문에 직접 바이너리를 실행하기보다 `cargo run` 사용을 권장합니다.)

## 조작법

- **마우스 좌클릭**: 기물 선택 → 강조된 칸을 클릭해 이동
  - 초록 원형 점: 이동 가능한 빈 칸
  - 초록 링(원형 테두리): 잡을 수 있는 상대 기물
  - 노랑 칸: 선택한 기물 / 마지막 수
  - 빨강 강조(킹): 체크 상태
- **U**: 한 수 되돌리기 (또는 오른쪽 패널의 "되돌리기" 버튼)
- **R**: 새 게임 시작
- **Esc**: 메뉴로 돌아가기 (또는 좌상단 "← 메뉴" 버튼)

오른쪽 **이동 기록** 패널에 대수기보(SAN: `e4`, `Nf3`, `O-O`, `exd5`, `e8=Q`, 체크 `+`, 체크메이트 `#`)가 순서대로 표시됩니다.

## 구현된 규칙

- 모든 기물의 정상 이동 / 캡처
- 백·흑 턴 교대
- 체크 / 체크메이트 / 스테일메이트 판정
- 자기 킹이 체크되는 수 금지(핀, 회피)
- 캐슬링(킹사이드·퀸사이드)
- 앙파상(en passant)
- 폰 승급(자동으로 퀸 승급)

규칙 엔진은 표준 perft 값(시작 위치 기준 1수 20, 2수 400, 3수 8902)으로 검증합니다.

```bash
cargo test
```

## 프로젝트 구조

```
src/
├── main.rs          # 앱 진입점, 상태(Menu/InGame/ComingSoon), 카메라/폰트
├── menu.rs          # 시작 메뉴 및 "AI 대전 준비 중" 화면
└── chess/
    ├── mod.rs       # 게임 플러그인, 공용 컴포넌트/리소스/좌표 변환
    ├── rules.rs     # 순수 체스 규칙 엔진(이동 생성·합법성·종료 판정·SAN) + 테스트
    ├── setup.rs     # 보드 타일 / 기물 텍스처 로드 / HUD·기록 패널 구성
    └── play.rs      # 입력, 렌더링(원형 이동표시), 상태/기록 표시, 되돌리기
assets/
├── fonts/          # 한글 폰트(NanumGothic)
└── pieces/         # 기물 이미지 12종 (wK.png, bQ.png, ...)
```

논리적인 보드 상태(`Board` 리소스)가 진실의 원천이며, 한 수가 둬질 때마다
시각 요소(기물·하이라이트)를 다시 그립니다. 규칙 엔진(`rules.rs`)은 Bevy 와
완전히 분리되어 있어, 추후 AI 대전 모드를 추가할 때 그대로 재사용할 수 있습니다.

## 앞으로 (AI 대전)

`AI 대전` 메뉴는 현재 "준비 중" 안내 화면으로 연결됩니다.
규칙 엔진의 `all_legal_moves` / `apply_move` / `status_for` 를 사용하면
미니맥스(알파-베타) 기반 AI 를 비교적 쉽게 얹을 수 있습니다.
